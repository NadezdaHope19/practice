# Часть 1. Вопросы для уточнения требований
## Бизнесу:
1.	Предполагаемые сроки реализации основных функций приложения, расставить приоритеты
2.	Какие экраны и разделы должны быть минимально (личный кабинет селлера/блогера, карточка рекламной кампании)?
3.	Какие данные должны быть доступны без интернета (например: заполнение личного кабинета, формирование карточки рекламной кампании) и как осуществлять синхронизацию при появлении сети?
4.	Какие платежные системы планируете подключать (СБП, SberPay, T-Pay, ЮKassa, Robokassa, ЮMoney и тп)?
5.	Нужно ли добавить к ключевым действиям селлера – подтвердить профиль, добавить платежную информацию, оплатить выполнение услуги?
6.	Какие статусы рекламной кампании предусмотреть (например: активная, на исполнении, выполнено, получена/переведена оплата)?
7.	Как формируется рейтинг селлера/блогера? Должны ли быть предустановленные критерии оценки (скорость выполнения заказа, качество контента, качество продукции по бартеру, своевременность оплаты компании)?
8.	При написании отзыва должна ли система определять, было ли взаимодействие у сторон, чтобы идентифицировать реальные отзывы.
9.	Предполагаемый размер текстового отзыва
10.	Нужна ли страница «топ селлеров», «топ блогеров»? По каким параметрам формировать? (оценка, количество выполненных заказов, количество осуществленных сделок)
11.	Какую нагрузку на приложение рассчитывать? Какая статистика по количеству пользователей сайта, планируется ли продвижение приложения для привлечения пользователей?
12.	Нужно ли осуществлять логирование действий пользователей. Какие?
13.	Какие способы идентификации планируете использовать при регистрации (проверка документов (KYC), авторизация через государственные или банковские сервисы (госулуги, sberid и тп), проверка юридического лица (для B2B))
14.	Если идентификация по документам (сканы паспорта, селфи), загружаемые в приложение, хранение на защищенных облачных серверах компании или в ЦОД? Какие требования предъявляются к хранению персональных данных, необходимость шифрования и тд.
15.	Должна ли быть валидация пользователя по соцсетям, для избежания дублей аккаунтов по разным номерам телефонов. Один блогер – один аккаунт в приложении.
16.	Нужна ли биометрическая авторизация (FaceID, отпечаток пальца)?
17.	Использовать дизайн-макеты web-версии или создавать новые?
18.	Планируется ли адаптация под темную/светлую тему?

## Блогеру:
19.	Как отображать список прилетающих офферов? Нужен ли функционал фильтрации (бартер, оплата) или сортировки (по дате поступления, по сумме и т.д.). Уточнить какой.
20.	Указать критерии для формирования оценки селлера (качественный продукт, своевременная оплата, скорость коммуникации и т.п.)
21.	Должен ли быть настраиваемый фильтр, какие принимать заказы. Например только оплата или бартер, от какой суммы рассматриваете сотрудничество? чтобы не получать нерелевантные предложения и не участвовать в поиске
22.	Нужна ли минимальная аналитика по заказам за период, если да, то какая?
23.	Нужна ли функция «Пожаловаться на селлера», для проверки добросовестности заказчика

## Селлеру:
24.	Как отображать список прилетающих офферов? Нужен ли функционал фильтрации или сортировки (по количеству подписчиков, по охвату и т.д.). Уточнить какой.
25.	 Указать критерии для формирования оценки блогера (скорость, качество контента и т.п.)
26.	Уточнить критерии поиска блогера на рекламную кампанию (страна/город проживания, количество подписчиков, тематика контента в соцсетях, пол, возраст и тп)
27.	Нужна ли функция «Пожаловаться на блогера», для проверки добросовестности исполнителя
28.	Нужна ли функция «Повторить сотрудничество»
